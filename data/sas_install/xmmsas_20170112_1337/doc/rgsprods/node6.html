XMM product format

XMM products adhere to a standard naming scheme (see [1]) which may be summarised by the following template: 

Pooooooooooiiteeennnnnndddd.xxx

The fields have the following meanings:

  • oooooooooo: 10-digit observation ID.

  • ii: instrument identifier (eg R1, R2).

  • t: exposure type (usually either `S' or `U', for `scheduled' or `unscheduled').

  • eee: 3-digit exposure ID.

  • nnnnnn: 6-character product type identifier.

  • dddd: 4-digit utility number.

  • xxx: file data type identifier. Examples are `FIT', `GIF', `PNG', `PDF'.
